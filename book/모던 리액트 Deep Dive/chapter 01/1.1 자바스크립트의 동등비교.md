# 1.1 자바스크립트의 동등 비교

리액트에서 성능 최적화의 목적은 대개 불필요한 리렌더링을 줄이기 위해 이루어진다.
이때 리렌더링이 발생하는 원인이 props의 동등 비교에 따른 결과이기에,
React 개발을 위해, 동등비교(얕은 비교)가 무엇인지 알아야한다.

**리액트의 가상DOM과 실제 DOM의 비교, 리액트 컴포넌트가 렌더링할지를 판단하는 방법, 변수나 함수 메모이제이션 등 모든 작업은 자바스크립트의 동등 비교를 기반으로 한다.**

## 1.1.1 자바스크립트의 데이터 타입

**원시타입 (primitive type)**

- boolean
- null
- undefined
- number
- string
- symbol
- bigint

**객체타입 (object/reference type)**

- object

### 원시타입

- 객체를 제외한 모든 타입 -> 객체가 아니므로 메서드를 갖지 않음
- null의 반환값은 object
- undefined는 '선언되었지만 할당되지 않은 값' / null은 '비어있는 값'

### 객체타입

- 배열, 함수, 정규식, 클래스 등
- 참조를 전달하는 참조타입으로도 불림

## 1.1.2 값을 저장하는 방식의 차이

### 불변값 vs 가변값

- 불변값

```javascript
let a = 20;
a = 10;

console.log(a); // 10
```

이 과정은

1. 메모리 공간 확보 후 다른 메모리 공간에 20할당
2. 20이 저장된 주소 a에 할당
3. 10 할당시 20이 저장된 메모리 공간은 그대로 두고, 10을 저장하는 메모리 공간 추가로 확보 후 저장
4. 그 공간의 주소를 a가 저장

-> 20이 저장되어 있는 주소는 그대로 두고, 10을 위한 주소 공간을 새로 할당. 따라서 20이라는 값은 변하지 않고 그대로 저장되어 있다.

- 가변 값

```java
var obj = {
    a: 10,
    b: 'abc',
}
var obj.a = 20;
```

-> obj의 프로퍼티 값을 재할당 하게 되면, obj의 메모리 주소는 변하지 않고, obj.a가 가진 주소가 변경된다. 즉, 새 객체가 만들어지는 것이 아니라 기존 객체의 내부 값만 변경
